Aide et tutoriel

Tout ce que fait ce navigateur, section par section.
Prise en main

Le bouton ⋮ a droite de la barre d'adresse ouvre le menu. Tout passe par la : il n'y a volontairement aucun bouton flottant sur les pages.

  • Page — analyser, code source, mode lecture, defilement infini, partage
  • Recherche — moteur par defaut, filtres editoriaux, sources
  • Confidentialite — navigation privee, niveau de protection, DNS, Tor
  • Scripts et styles — scripts utilisateur, feuilles CSS, commandes
  • Favoris — ouvrir, ajouter, supprimer

Le chiffre vert a gauche de la barre d'adresse compte les elements bloques. Un appui le desactive, un appui long affiche le total.

Historique nouveau

Menu › Historique conserve les pages consultees, et permet de les retrouver par titre, par site, ou par leur contenu.

Deux niveaux, volontairement separes. L'adresse et le titre sont conserves par defaut, comme dans tout navigateur. L'indexation du texte des pages, elle, est desactivee par defaut : c'est ce qui transforme « j'ai lu un truc la-dessus il y a trois semaines » en une simple requete, mais c'est aussi une capacite qui merite d'etre choisie plutot que subie.

Une fois activee, la recherche parcourt titres, adresses et contenus, et affiche un extrait avec le terme surligne. La comparaison ignore les accents : « refere » trouve « refere » comme « refere ».

Reglages dans le bouton du meme nom : activation, indexation du texte, et liste de sites a ne jamais enregistrer — utile pour une banque ou une messagerie.

La navigation privee n'est jamais enregistree, quels que soient ces reglages. L'historique reste sur l'appareil, n'est pas inclus dans la synchronisation, et disparait avec Confidentialite › Effacer toutes les donnees. Huit cents pages au maximum, les plus anciennes cedant la place ; le texte est tronque a huit mille caracteres par page.

Un point a peser : indexer ce que vous lisez rend l'appareil plus revelateur qu'un simple historique d'adresses. Quiconque y a acces peut fouiller le contenu de vos lectures. C'est utile, mais ce n'est pas anodin.

Flux maison nouveau

Beaucoup de sites ont abandonne les flux RSS. Celui-ci en fabrique un a partir de n'importe quelle page qui liste des articles.

Page › Creer un flux ouvre le pointeur : vous touchez un seul article, le navigateur reconnait les autres par leur structure commune. Le compteur affiche combien d'entrees sont visees, ce qui permet de verifier avant de valider. « Parent » remonte d'un cran si vous n'avez attrape que le titre au lieu du bloc entier.

Titre et lien sont deduits automatiquement : un intitule structure d'abord, le texte du lien a defaut. Une date est reprise si la page en expose une.

Menu › Mes flux affiche les entrees, les nouveautes signalees par un liseré vert. Chaque flux se releve pendant que le navigateur tourne, toutes les trois heures par defaut. Les notifications sont desactivees par flux : un releve de titres deviendrait vite envahissant, a la difference d'une surveillance ponctuelle.

Chaque flux s'exporte au format RSS dans vos telechargements, lisible par n'importe quel agregateur exterieur.

Les entrees presentes a la creation sont marquees comme deja vues : seules les suivantes apparaitront en nouveaute. Si le site refond sa mise en page, le selecteur peut cesser de correspondre — recreez alors le flux.

File de lecture hors ligne nouveau

Page › Lire plus tard met l'article de cote. Contrairement a un simple marque-page, le contenu est reellement conserve : il reste lisible sans reseau, dans le metro comme en avion, et meme si la page d'origine disparait.

L'article est nettoye a l'enregistrement — scripts, cadres, formulaires, encarts de partage, blocs de commentaires et suggestions sont retires. Ne reste que le texte, les titres, les listes, les citations et les images.

Les images sont incorporees par defaut, ce qui rend la lecture complete hors ligne. Les decocher divise nettement la place occupee : utile pour un long dossier illustre dont seul le texte vous interesse.

Menu › File de lecture ouvre la liseuse : typographie serif confortable, filtrage par titre ou par site, bouton « non lus », marquage lu, et retour a la page d'origine d'un geste.

Soixante articles au maximum, les plus anciens cedant la place. Le contenu vit dans le stockage du navigateur : Confidentialite › Effacer toutes les donnees le supprime egalement. Pensez a la synchronisation Git si vous tenez a le conserver.

Comparaison de versions nouveau

Archiver le texte d'une page aujourd'hui, revenir dans un mois, et voir exactement ce qui a bouge. Utile pour reperer une correction non signalee, un passage retire, un chiffre revu.

Page › Archiver cette version enregistre l'etat courant. Page › Comparer avec l'archive confronte la page affichee a la derniere version conservee.

Le resultat se lit d'un coup d'oeil : les ajouts sur fond vert, les suppressions barrees sur fond rouge, et les passages inchanges masques sauf lorsqu'ils bordent un ecart, pour situer le contexte.

Deux choix de conception valent d'etre connus. Seul le texte principal est retenu — menus, encarts et pieds de page sont ecartes, sinon le moindre bandeau tournant produirait un ecart a chaque comparaison. Et seul le texte est conserve, pas la page entiere : c'est ce qui se compare utilement, et cela tient en quelques dizaines de kilo-octets par version.

Quatre versions au maximum par adresse, les plus anciennes cedant la place. Menu › Archives de pages liste le tout, avec export du texte de toutes les versions dans un fichier.

La comparaison porte sur des blocs entiers : deplacer un paragraphe apparait comme une suppression suivie d'un ajout. Sur un document tres long, une limite de securite remplace le detail par un message plutot que de saturer la memoire du telephone.

Procedes trompeurs nouveau

Page › Procedes trompeurs examine la page et signale huit motifs connus de conception manipulatrice.

  • Cases deja cochees — seules celles dont l'intitule evoque une offre ou un consentement sont retenues ; une case cochee n'a rien de suspect en soi
  • Compte a rebours — rechargez la page : s'il repart au meme point, il ne mesure aucune echeance
  • Mentions de rarete — « plus que deux en stock », « derniers exemplaires »
  • Compteurs d'audience — « douze personnes regardent ce produit »
  • Refus culpabilisant — un refus formule pour mettre mal a l'aise
  • Bandeau sans refus visible — un consentement demande sans que le refus soit propose au premier niveau
  • Texte tres reduit — mentions sous dix pixels
  • Fermeture reduite — croix de moins de vingt-deux pixels sur une fenetre superposee

Montrer fait defiler jusqu'a l'element et l'entoure. Neutraliser, la ou c'est possible, decoche les cases, masque le decompte, agrandit les textes minuscules et les zones de fermeture.

Ce sont des indices, pas des verdicts. Un compte a rebours peut etre authentique, un stock reellement limite, une case avoir ete cochee par vous un instant plus tot. La detection repose sur des formulations et des mesures : elle se trompe dans les deux sens, et l'absence de signal ne garantit rien.

Qui parle a qui nouveau

Page › Qui parle a qui repond a une question simple : en ouvrant cette page, a qui votre appareil a-t-il parle ?

Le rapport commence par une phrase en clair — nombre de domaines tiers contactes, combien sont des regies ou des traqueurs, combien le blocage en a coupe. Vient ensuite le detail, un domaine par ligne, classe par nombre de requetes : volume transfere, nature des echanges (scripts, images, appels d'API), et sous-domaines rencontres.

Trois etiquettes qualifient chaque tiers :

  • regie ou traqueur — le domaine figure dans les listes publicitaires
  • categorie — il releve d'une de vos categories de filtrage
  • proprietaire — le groupe auquel il appartient, quand la carte de publishers.js le connait

C'est ce dernier point qui rend l'outil interessant : voir qu'une page d'apparence anodine dialogue avec quatre domaines appartenant au meme groupe en dit plus long qu'une liste d'adresses techniques.

Le journal se remplit au fil du chargement, et ne remonte pas avant l'ouverture du navigateur sur cette page. Pour un releve complet, rechargez la page puis ouvrez le rapport.

Surveillance de pages nouveau

Le navigateur revient verifier un element que vous avez designe, et vous previent quand il change. Prix, disponibilite, nouvelle publication, correction discrete d'un article.

Creer une surveillance. Ouvrez la page, puis Page › Surveiller un element. Touchez l'element, « Parent » remonte d'un cran si vous n'avez attrape que le texte. La barre affiche en direct la valeur qui sera retenue comme reference.

Trois modes de declenchement :

  • Le texte change — toute modification du contenu
  • Le nombre change — seul le premier nombre est retenu, ce qui ignore les variations de mise en forme autour d'un prix
  • Apparition ou disparition — pour un bouton « en stock », un bandeau, une mention qui surgit

Suivi dans Menu › Surveillances : valeur actuelle, valeur precedente barree, date du dernier changement, selecteur employe. « Reference actuelle » repart de la valeur presente une fois le changement pris en compte. « Verifier maintenant » force un passage immediat.

Les verifications ont lieu pendant que le navigateur tourne : un rattrapage huit secondes apres le demarrage, puis un passage toutes les cinq minutes pour les surveillances dont l'echeance est atteinte. Une surveillance continue application fermee demanderait un service permanent en arriere-plan, avec le cout en batterie correspondant — un compromis que je n'ai pas voulu imposer.

Le selecteur peut cesser de correspondre si le site refond ses pages. La valeur devient alors vide : reprenez le pointeur pour la recalculer.

Raccourcis de l'accueil nouveau

Sous la barre de recherche, une grille de pastilles menant a vos sites habituels. Le bouton + en ajoute un ; Page › Ajouter aux raccourcis enregistre la page ouverte sans avoir a recopier son adresse.

Un appui long sur une pastille propose de la renommer, d'en changer l'icone, de la deplacer ou de la retirer. L'icone accepte une lettre ou une emoji ; laissee vide, c'est l'initiale du nom qui s'affiche.

Les pastilles sont des aplats colores, non des favicons. Afficher de vraies icones supposerait d'interroger chaque site a l'ouverture de l'accueil, donc de signaler votre presence a une dizaine de serveurs avant meme que vous ne tapiez quoi que ce soit. La couleur est calculee a partir de l'adresse : elle reste stable sans rien demander a personne.

Les raccourcis accompagnent la page de marque : ils disparaissent pendant une recherche, et ne s'affichent pas si vous avez choisi un moteur externe.

Demarrage de l'application

Au lancement, le logo s'affiche sur fond sombre pendant que le moteur et l'extension se chargent, puis la page de recherche apparait.

Trois reglages evitent les a-coups visuels. Le theme de la fenetre est sombre des l'ouverture, avant meme que l'application ne dessine. Le compositeur de Gecko peint dans la meme couleur, alors qu'il utilise du blanc par defaut. Et l'ecran de demarrage ne se retire qu'au premier rendu effectif du contenu, pas a la fin du chargement reseau — ce dernier survient trop tot, quand rien n'est encore affiche.

Si l'extension tarde a repondre, un repli au bout de cinq secondes ouvre quand meme une page, pour ne jamais rester bloque sur le logo.

Onglets nouveau

Le compteur encadre en vert, a droite de la barre d'adresse, indique le nombre d'onglets ouverts. Un appui ouvre la liste, un appui long cree directement un nouvel onglet.

La liste montre le titre de chaque page et son adresse, avec un reperage : disque plein pour l'onglet visible, cercle vide pour les autres, demi-disque pour un onglet prive. On y trouve aussi la creation d'un onglet normal ou prive, la fermeture de l'onglet courant, et la fermeture de tous les autres.

Un lien ouvert par le site dans une nouvelle fenetre — target="_blank" — cree desormais un onglet en arriere-plan au lieu de remplacer la page courante.

Le bouton retour suit l'ordre attendu : d'abord l'historique de la page, puis la fermeture de l'onglet s'il en reste d'autres, et seulement ensuite la sortie de l'application. Le dernier onglet n'est jamais ferme : il revient a l'accueil.

Restauration. Les onglets ouverts sont retrouves au lancement suivant, dans la limite de douze. Les onglets prives en sont exclus : ils ne laissent aucune trace, c'est leur raison d'etre. Le comportement se desactive dans Confidentialite › Restaurer les onglets au demarrage.

Chaque onglet garde sa page en memoire. Sur un telephone modeste, au-dela d'une dizaine d'onglets charges, le systeme peut fermer l'application pour recuperer de la memoire — la restauration les rouvrira, mais la navigation en cours sera perdue.

Widgets d'ecran d'accueil nouveau

Deux widgets a poser depuis l'ecran d'accueil, par appui long puis « Widgets ».

Barre de recherche (4 x 1) : la zone principale ouvre le navigateur avec le clavier deja actif sur la barre d'adresse. Deux icones a droite lancent directement la navigation privee et la liste des favoris.

Compteur de blocage (2 x 2) : nombre d'elements bloques, pastille verte ou grise selon l'etat, et un appui bascule le blocage.

L'etat du blocage est desormais conserve d'un lancement a l'autre : si vous le coupez depuis le widget, il reste coupe au demarrage suivant.

Le compteur reflete la derniere valeur transmise par le navigateur, rafraichie au plus toutes les quatre secondes pour ne pas solliciter l'ecran d'accueil inutilement. Il n'avance donc pas quand l'application est fermee.

Blocage des publicites et traqueurs

Trois couches travaillent ensemble. La protection anti-pistage de Gecko en mode strict, la meme que Firefox. Une extension qui coupe les requetes vers environ 150 domaines embarques, completes au demarrage par les listes d'AdAway et de Peter Lowe, rafraichies toutes les 24 heures. Et un filtrage cosmetique qui masque les emplacements publicitaires avant le premier rendu.

Les images bloquees sont remplacees par un pixel transparent, ce qui evite de casser les mises en page.

Si un site casse, le premier reflexe est d'ajouter son domaine dans Recherche › Filtres › Domaines a ne jamais filtrer.

Filtres editoriaux et categories

Huit categories de sites, activables separement dans Recherche › Filtres : publicite, medias du groupe Bollore, medias d'extreme droite, multinationales, reseaux sociaux, football, sport, resolveurs DNS chiffres.

Les domaines de chaque categorie sont dans des fichiers texte modifiables, un domaine par ligne, sous app/src/main/assets/adblock/lists/. Vous les editez, vous recompilez, c'est tout.

La liste Bollore repose sur la propriete du capital, verifiable. La liste « extreme droite » est un classement editorial, conteste par plusieurs des titres concernes. Les deux sont modifiables : c'est votre filtre, pas le mien.

Une categorie filtree agit a trois endroits : les resultats de recherche, les liens dans les pages, et la navigation elle-meme si vous cochez l'option.

Moteur de recherche

Deux reglages distincts, souvent confondus.

Le moteur de la barre d'adresse se choisit dans Recherche › Moteur : metamoteur integre, DuckDuckGo, Qwant, Ecosia, Brave, Startpage, Mojeek, Marginalia, Wikipedia, Google, Bing, ou un modele personnalise ou %s remplace la requete.

Les sources du metamoteur se cochent dans Recherche › Filtres › Sources. Le metamoteur interroge plusieurs moteurs en parallele, fusionne les resultats et remonte ceux sur lesquels ils s'accordent.

Raccourcis facon bangs : !w Wikipedia, !yt YouTube, !gh GitHub, !osm OpenStreetMap, !mp Mediapart, !lm Le Monde.

Cookies et consentement

Masquer un bandeau ne refuse rien : sans action, beaucoup de sites considerent l'absence de reponse comme un accord. Le navigateur appelle donc directement les API de refus des plateformes de consentement — Didomi, OneTrust, Cookiebot, Usercentrics, Axeptio, tarteaucitron, Quantcast et une dizaine d'autres — puis retombe sur le clic du bouton s'il le faut.

Les cookies tiers sont refuses a la reception, et ne sont pas renvoyes. Une liste d'exemptions protege ce qui casserait sinon : connexions Google, FranceConnect, Stripe, PayPal, captchas.

Bouton de purge immediate et option de purge a la fermeture dans Recherche › Filtres › Cookies.

Analyse de page

Page › Analyser la page ouvre cinq onglets.

Ressources — douze categories filtrables, plus un champ d'extensions libre. Tapez webp, avif et vous n'avez plus que celles-la. Chaque entree s'ouvre, se copie, se telecharge. Selection multiple, telechargement groupe, export de la liste en fichier texte, et mesure des tailles par requetes HEAD.

Code — « DOM actuel » montre la page apres execution du JavaScript, « Source d'origine » retelecharge le HTML brut du serveur. La difference entre les deux est souvent instructive. Recherche avec surlignage et compteur.

Console — voir la section suivante.

Reseau — voir la section suivante.

Extraire — transforme la page en tableau exportable, section dediee plus bas.

Infos — doctype, encodage, langue, nombre d'elements, domaines tiers, cookies lisibles, stockage local, et toutes les balises meta.

Console JavaScript nouveau

Onglet Console de l'analyseur. Saisissez une expression, appuyez sur Executer. La fleche remonte l'historique.

Deux contextes, commutables par le bouton en haut :

  • page — le code s'execute avec les variables du site, comme dans les outils de developpement d'un navigateur de bureau. Soumis a la politique de securite du site : certains la refusent, un message vous le dit.
  • isole — toujours disponible, quel que soit le site. Les objets du site restent accessibles via unsafeWindow.

Raccourcis fournis dans les deux modes : $('sel') pour un element, $$('sel') pour un tableau. Les promesses sont attendues automatiquement, les elements DOM affiches en HTML.

Les messages emis par la page — console.log, avertissements, erreurs, promesses rejetees — sont captures et affiches au fil de l'eau.

La capture demarre au chargement de l'extension : les tout premiers messages d'une page peuvent manquer. Rechargez avec la console deja ouverte pour les voir.

Journal reseau nouveau

Onglet Reseau de l'analyseur : les 250 dernieres requetes de la page, avec statut, methode, type, type MIME, taille, duree.

Le filtre par type est la fonction la plus utile. Choisissez xmlhttprequest et vous ne voyez plus que les appels d'API du site — le bouton « Reponse » affiche alors le JSON renvoye. C'est la facon la plus directe de comprendre comment un site charge ses donnees.

Les requetes coupees par vos filtres sont marquees bloque, ce qui permet de verifier ce que le blocage fait reellement, et de reperer un faux positif quand un site se comporte mal.

Export en CSV pour analyse ailleurs.

Extracteur structure nouveau

Onglet Extraire de l'analyseur : transforme une page en tableau. Vous designez l'element repete — une carte de produit, un resultat, une ligne — puis les colonnes a en tirer.

Deviner cherche le conteneur dont les enfants se ressemblent le plus et en deduit le selecteur. Pointer vous laisse toucher l'element directement sur la page ; le compteur indique combien d'elements le selecteur attrape, ce qui permet de verifier qu'il vise bien toute la liste et pas une seule ligne. Colonnes auto propose titre, lien, image et prix a partir de la premiere ligne trouvee.

Chaque colonne prend un nom, un selecteur relatif a la ligne — vide pour viser la ligne entiere — et le type de valeur : texte, lien, image, HTML, ou un attribut nomme. Les liens et images sortent en adresses absolues.

Le champ Pages a suivre enchaine la pagination en reutilisant la meme detection que le defilement infini : lien rel="next", element suivant dans la pagination, libelles, ou incrementation du parametre d'URL. Les pages suivantes sont recuperees et analysees sans quitter celle-ci.

Export en CSV ou JSON dans les telechargements, ou copie du CSV dans le presse-papiers. L'apercu montre les premieres lignes au fil de la saisie.

Les pages qui construisent leur contenu en JavaScript apres chargement ne se pretent pas au suivi de pagination : la page courante s'extrait bien, les suivantes reviennent vides. Dans ce cas, laissez « Pages a suivre » a 1 et faites defiler manuellement.

Telechargements et extraction audio

Depuis l'onglet Ressources : bouton par ligne, ou telechargement groupe de tout ce que vos filtres affichent. Au-dela de 25 fichiers, une confirmation s'affiche.

Les videos et fichiers audio ont un bouton Extraire l'audio. La piste est recopiee sans reencodage — AAC vers .m4a, Opus vers .ogg — et deposee dans le dossier Musique. C'est instantane et sans perte.

Android ne fournit pas d'encodeur MP3, seulement des decodeurs. Le .m4a produit est lu par tout ce qui lit du MP3. Les flux segmentes .m3u8 et .mpd ne sont pas pris en charge.

Sauvegarde en un fichier nouveau

Page › Enregistrer en un fichier produit un .html autonome dans vos telechargements : styles, images, polices et icones sont incorpores dans le fichier lui-meme. Il reste lisible hors ligne et peut etre deplace ou archive sans rien perdre.

Deux options. Les images et polices peuvent etre exclues, ce qui donne un fichier tres leger utile pour archiver un texte. Les scripts sont retires par defaut : les conserver rend l'archive fragile et lui redonne acces au reseau.

Les feuilles de style sont toujours incorporees, avec resolution des @import et des url() qu'elles contiennent — sans cela, l'archive perdrait sa mise en page et ses images de fond.

Un bandeau discret en haut du fichier rappelle l'adresse d'origine, la date d'enregistrement et le volume incorpore. Votre feuille CSS personnelle du site, si vous en avez une, est appliquee a l'archive.

Limites assumees : 2 Mo par ressource et 12 Mo au total, pour ne pas produire un fichier ingerable sur telephone. Les videos, cadres et contenus externes ne peuvent pas etre incorpores : ils sont remplaces par une mention indiquant leur adresse d'origine.

Scripts utilisateur

Scripts et styles › Mes scripts : editeur integre avec modele prerempli, interrupteur par script, import depuis une URL, controle de syntaxe avant enregistrement.

En-tetes reconnus : @match, @include, @exclude, @run-at, @require, @noframes, @grant. Ajoutez @world page pour executer dans le contexte du site plutot que dans le bac a sable.

API disponible : GM_getValue, GM_setValue, GM_deleteValue, GM_listValues, GM_addStyle, GM_registerMenuCommand, GM_xmlhttpRequest, GM_openInTab, GM_setClipboard, GM_info, unsafeWindow.

GM_xmlhttpRequest passe par l'extension : pas de restriction CORS, vos scripts peuvent interroger n'importe quel domaine.

Les commandes enregistrees apparaissent dans Scripts et styles › Commandes.

Un script utilisateur a tous les droits sur les pages qu'il touche : cookies, formulaires, contenu. N'importez que du code dont vous comprenez l'effet.

Styles CSS par site nouveau

Scripts et styles › Mes styles CSS : des feuilles appliquees automatiquement selon le site. Le complement naturel des scripts — le JavaScript pour le comportement, le CSS pour l'apparence.

Motifs acceptes, un par ligne :

  • exemple.fr — le domaine et ses sous-domaines
  • *.exemple.fr — les sous-domaines seuls
  • exemple.fr/blog* — restreint au chemin
  • /expression/ — expression reguliere sur l'adresse complete
  • * — toutes les pages

Cinq modeles fournis dans l'editeur : masquer un element, elargir le contenu, fond sombre, confort de lecture, suppression des bandeaux colles. Les accolades sont verifiees avant enregistrement.

Les modifications s'appliquent a chaud : une page deja ouverte se met a jour sans rechargement.

Le pointeur. Page › Masquer un element passe la page en mode designation : touchez un element, il se surligne en vert et la barre du bas affiche le selecteur calcule ainsi que le nombre d'elements vises. « Bloc parent » remonte d'un cran quand vous n'avez attrape que le texte au lieu du cadre entier. « Masquer » enregistre la regle pour ce site, « Editer » ouvre l'editeur avec le selecteur pre-rempli.

Page › CSS de ce site ouvre directement la feuille du site courant, ou en cree une.

Le selecteur privilegie l'identifiant, puis les classes qui paraissent stables, en ecartant celles qui ressemblent a du genere automatiquement. Sur un site qui regenere ses classes a chaque deploiement, une regle peut cesser de fonctionner : reprenez le pointeur pour la recalculer.

Defilement infini

Sur un site pagine, les pages suivantes s'ajoutent automatiquement quand vous approchez du bas. La detection essaie, dans l'ordre : le lien rel="next", l'element suivant dans la pagination, les libelles de lien (« suivant », « next », »), puis l'incrementation du parametre d'URL (page=, offset=, /page/2/).

Reglages dans Recherche › Filtres › Navigation : mode automatique ou bouton « Charger la suite », et limite de pages enchainees. Bascule par site dans Page › Defilement infini ici.

Sur les applications web qui chargent deja en JavaScript, il n'y a pas de lien a detecter : le script ne fait rien.

Redirections vers les facades nouveau

Certains sites sont remplaces avant meme le chargement par une facade libre : meme contenu, sans publicite, sans traqueur, sans script de mesure, et souvent sans mur de connexion.

  • YouTube → Invidious (desactive par defaut)
  • Twitter / X → Nitter
  • Reddit → Redlib
  • Medium → Scribe
  • TikTok → ProxiTok
  • Imgur → rimgo
  • Fandom → BreezeWiki
  • Quora → Quetre
  • Stack Overflow → AnonymousOverflow
  • IMDb → libremdb
  • Google Traduction → Lingva
  • Wikipedia → Wikiless (desactive par defaut)

La traduction du chemin est geree service par service : youtu.be/ID et /shorts/ID deviennent /watch?v=ID, un blog auteur.medium.com devient /@auteur, un wiki jeu.fandom.com devient /jeu.

Une option separee remonte aux articles d'origine depuis les pages accelerees de Google, qui masquent l'editeur reel derriere une adresse Google.

Reglages dans Confidentialite › Redirections : activation par service, choix de l'instance, ou saisie de la votre.

Le cas de YouTube est particulier : la plateforme bloque activement les instances Invidious publiques, qui affichent alors une page d'erreur au lieu des resultats. La redirection est donc desactivee par defaut. Vous pouvez l'activer et essayer une instance : certaines tiennent quelques semaines. Le blocage publicitaire du navigateur reste actif sur YouTube, mais il ne remplace pas ce qu'apportait la facade — sans elle, YouTube associe de nouveau vos visionnages a votre compte et a votre adresse.

Plus generalement, les instances publiques sont tenues par des benevoles : elles tombent, se saturent, disparaissent. Si une facade ne repond plus, changez-en dans la liste. Deux echappatoires : ajoutez #direct a la fin d'une adresse pour la charger sans redirection, ou utilisez Page › Ne plus rediriger ce service depuis la facade elle-meme, ce qui desactive le service et vous renvoie a l'original.

Confidentialite

Trois niveaux dans Confidentialite › Niveau, chacun annonce ce qu'il casse avant d'etre applique.

  • Standard — cloisonnement des cookies par site, referent tronque, capteurs coupes. Rien ne casse.
  • Renforce — ajoute l'uniformisation de l'empreinte : agent, langue, fuseau, taille d'ecran, canvas. Heure en UTC, langue annoncee en anglais, contenus DRM indisponibles.
  • Strict — bandes grises de normalisation, WebGL, WebAudio et WebAssembly coupes, polices du site ignorees. Des applications web cesseront de fonctionner.

S'y ajoutent la navigation privee, l'effacement complet des donnees, et le DNS chiffre au choix entre Quad9, Cloudflare, Mullvad et dns0.eu.

Changer de niveau relance l'application : Gecko ne lit ses preferences qu'au demarrage.

Identite de l'appareil nouveau

Page › Identite de l'appareil : neuf profils prets a l'emploi — telephone et tablette Android, iPhone, iPad, PC Windows sous Firefox ou Chrome, Mac sous Safari, PC Linux — plus une chaine personnalisee.

Le remplacement agit sur trois plans a la fois, et c'est ce qui compte :

  • l'en-tete HTTP envoyee a chaque requete
  • navigator.userAgent lu par le JavaScript de la page
  • la mise en page, qui bascule en largeur ordinateur ou telephone

L'extension aligne en plus platform, oscpu, appVersion, vendor et le nombre de points tactiles. Sans cela un site verrait un agent Windows sur une plateforme Linux aarch64 avec cinq points tactiles : la contradiction est facile a reperer et vous singularise davantage que de ne rien changer.

Deux limites a garder en tete. Le moteur reste Gecko : se declarer Chrome ou Safari ne change ni les fonctions disponibles ni le rendu, et un site qui teste les capacites plutot que l'agent verra la difference. Ensuite, aux niveaux renforce et strict, la protection anti-empreinte impose deja son propre agent — superposer un profil recree une incoherence. Verifiez le resultat dans Confidentialite › Diagnostic d'empreinte, qui signale precisement ce genre de contradiction.

Usage courant : forcer la version ordinateur d'un site qui impose une version mobile appauvrie, ou l'inverse.

Diagnostic d'empreinte nouveau

Confidentialite › Diagnostic d'empreinte mesure ce qu'un site peut lire de votre navigateur, et signale les valeurs qui vous singularisent.

Six familles sont examinees : identite declaree (agent, langue, plateforme), ecran et fenetre, fuseau horaire et format de date, materiel annonce, empreintes de rendu (canvas, WebGL, audio, polices reconnues), et enfin reseau et stockage, dont un test de fuite WebRTC qui verifie si votre adresse locale s'echappe.

Le rapport signale aussi les incoherences, qui sont elles-memes un signal distinctif : un agent annoncant un mobile sur un appareil sans point tactile, ou une protection anti-empreinte apparemment active alors que le canvas renvoie toujours un rendu propre a l'appareil.

La mesure s'execute dans le contexte de la page consultee, pas dans une page de l'extension : les protections ne s'appliquent pas de la meme facon aux deux, et seul ce contexte reflete ce qu'un site observe reellement. Ouvrez donc le diagnostic depuis un site ordinaire, pas depuis l'accueil.

Le nombre de points distinctifs n'est pas un score d'anonymat : il compte les valeurs qui sortent de l'ordinaire. Peu de points signifie que vous ressemblez a beaucoup de monde, et c'est precisement cela qui protege.

Usage pratique : lancez la mesure, passez au niveau strict dans Confidentialite › Niveau, relancez, comparez. Vous verrez concretement ce que chaque niveau masque.

Tor et adresses .onion

Necessite Orbot, qui fournit un proxy local. Activation dans Confidentialite › Tor. La resolution DNS se fait cote Tor, les adresses .onion sont accessibles, et WebRTC comme la prelecture DNS sont coupes car ils contourneraient le proxy.

Les telechargements et l'extraction audio passent egalement par Orbot : sans cela, ils reveleraient votre adresse IP reelle.

Ce mode donne l'acces au reseau Tor, pas l'anonymat de Tor Browser. L'anonymat vient de la ressemblance : Tor Browser protege parce que ses utilisateurs sont interchangeables. Ce navigateur est unique par construction — extensions rares, scripts personnels, listes de filtres sur mesure — et tout cela reste mesurable. Excellent contre le pistage commercial ; insuffisant si votre securite en depend.

Synchronisation par depot Git nouveau

Menu › Synchronisation depose vos reglages dans un depot GitHub a vous, sous forme d'un seul fichier JSON. Vous editez depuis un ordinateur, le telephone suit ; ou vous transferez tout vers un nouvel appareil en une manipulation.

Quatre actions : Envoyer televerse l'instantane courant, Recuperer restaure la version du depot, Comparer affiche les deux inventaires cote a cote sans rien modifier, et Enregistrer retient la configuration sur l'appareil.

Une case active la recuperation automatique au demarrage : le fichier n'est applique que s'il est plus recent que la derniere recuperation, ce qui evite d'ecraser en boucle. A l'envoi, si le depot contient une version plus recente que la votre, une confirmation vous previent avant d'ecraser.

Ce qui est synchronise : scripts utilisateur et leurs valeurs, feuilles CSS, categories de filtrage actives, domaines ajoutes ou exemptes, reglages de cookies, de recherche, de redirections, profil d'appareil, sites exclus du defilement infini, et favoris. Ce qui ne l'est pas : historique, cookies de session, cache — c'est-a-dire tout ce qui est propre a un appareil.

Un champ separe permet d'importer depuis une adresse brute quelconque, sans jeton ni compte. Pratique pour partager une configuration en lecture seule, ou pour reprendre celle d'un ami.

Le jeton reste sur l'appareil, dans le stockage de l'extension. Utilisez un jeton a portee restreinte : un seul depot prive, autorisation « Contents » en lecture et ecriture, rien d'autre. N'ecrivez jamais un jeton dans le code source du navigateur, qui est publie sur GitHub — le detecteur de secrets bloquerait d'ailleurs l'envoi.

Compilation et signature nouveau

Chaque envoi vers le depot declenche un controle statique avant la compilation : syntaxe JavaScript, coherence du manifeste, collisions de portee, methodes Java citees mais absentes, ressources manquantes, XML mal forme. Ces familles de pannes correspondent a celles reellement rencontrees ; elles arretent desormais le build en quelques secondes au lieu de le laisser echouer apres une minute.

Deux versions sont produites. L'APK de debug, automatique a chaque envoi, porte le suffixe .debug et cohabite avec l'autre. L'APK de release, declenche a la demande, est minifie, allege de ses ressources inutilisees, et signe avec votre propre cle.

Avant de compiler, le workflow verifie que le mot de passe ouvre bien le trousseau, et affiche la taille et l'empreinte du fichier reconstruit. Une erreur de secret est ainsi signalee en quelques secondes, au lieu d'echouer apres trois minutes de compilation.

La difference pratique tient a la signature : une version de debug doit etre desinstallee avant chaque mise a jour, ce qui efface favoris, scripts et reglages. Une version signee s'installe par-dessus la precedente.

La cle n'est jamais dans le depot : elle vit dans les secrets du depot GitHub, et le fichier temporaire est efface a la fin de la compilation. Sauvegardez le trousseau ailleurs que sur le telephone — le perdre interdit definitivement toute mise a jour de l'application installee. La marche a suivre est dans tools/SIGNING.md.
